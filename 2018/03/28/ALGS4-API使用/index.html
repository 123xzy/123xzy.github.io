ALGS4-API使用

Posted on 2018-03-28

ALGS4初认识

初看这本书的时候，最让人头疼的就是作者自己写的标准库，但是自己对Java什么都不知道，就照抄书上的代码，每次编译错误就很让人心烦，于是又踏上了学习Java的路。
Java一个最佳实践就是每个静态方法库都包含一个main()函数来测试库中的方法，而有些编程语言是不支持多个main()方法的。
模块化编程的一个重要部分就是记录库方法的用法并供其他人参考的文档。这一点深有体会，因为随着代码量的增加，回头看很多代码，如果没有适当的注释、文档提示，要看懂曾经的代码需要很大的经历。因此写注释、写文档不仅对他人参考很重要，对自己也同样的重要。

编写自己的库，作为日后一个可以重用的库

  • 编写用例
  • 明确API
  • 编写测试用例

API的目的是为将调用和实现分离，除了API给出的信息，调用者不需要知道其他实现的细节，调用者只需要遵守API给出的这份契约就可以编写复杂的应用程序。除此之外，实现的代码和调用者代码分离，日后要是对要更好的实现，就能轻易替换。
因此接下来将自己实现本书中用到的方法库，分别如下：

  • StdOut：标准静态输出库
  • StdIn：标准静态输入库
  • StdRandom：随机数静态库
  • StdStats：数据分析静态方法库
  • In和OUt：用于读取和写入数组的静态方法库
